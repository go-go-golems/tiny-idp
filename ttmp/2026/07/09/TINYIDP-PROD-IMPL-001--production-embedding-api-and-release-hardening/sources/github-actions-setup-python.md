<article><H2 dir="auto">setup-python</H2> <p dir="auto"><a href="https://github.com/actions/setup-python/actions/workflows/basic-validation.yml"><img src="https://github.com/actions/setup-python/actions/workflows/basic-validation.yml/badge.svg?branch=main" alt="Basic validation"></a> <a href="https://github.com/actions/setup-python/actions/workflows/test-python.yml"><img src="https://github.com/actions/setup-python/actions/workflows/test-python.yml/badge.svg?branch=main" alt="Validate Python e2e"></a> <a href="https://github.com/actions/setup-python/actions/workflows/test-pypy.yml"><img src="https://github.com/actions/setup-python/actions/workflows/test-pypy.yml/badge.svg?branch=main" alt="Validate PyPy e2e"></a> <a href="https://github.com/actions/setup-python/actions/workflows/e2e-cache.yml"><img src="https://github.com/actions/setup-python/actions/workflows/e2e-cache.yml/badge.svg?branch=main" alt="e2e-cache"></a></p> <p dir="auto">This action provides the following functionality for GitHub Actions users:</p> <ul dir="auto"> <li>Installing a version of Python or PyPy and (by default) adding it to the PATH</li> <li>Optionally caching dependencies for pip, pipenv and poetry</li> <li>Registering problem matchers for error output</li> </ul> <H2 dir="auto">Breaking changes in V6</H2> <ul dir="auto"> <li>Upgraded action from node20 to node24 <blockquote> <p dir="auto">Make sure your runner is on version v2.327.1 or later to ensure compatibility with this release. See <a href="https://github.com/actions/runner/releases/tag/v2.327.1">Release Notes</a></p> </blockquote> </li> </ul> <p dir="auto">For more details, see the full release notes on the <a href="https://github.com/actions/setup-python/releases/tag/v6.0.0">releases page</a></p> <H2 dir="auto">Basic usage</H2> <p dir="auto">See <a href="https://github.com/actions/setup-python/blob/main/action.yml">action.yml</a></p> <p dir="auto"><strong>Python</strong></p> <pre><code>steps:
- uses: actions/checkout@v6
- uses: actions/setup-python@v6
  with:
    python-version: '3.13'
- run: python my_script.py</code></pre> <p dir="auto"><strong>PyPy</strong></p> <pre><code>steps:
- uses: actions/checkout@v6
- uses: actions/setup-python@v6
  with:
    python-version: 'pypy3.10'
- run: python my_script.py</code></pre> <p dir="auto"><strong>GraalPy</strong></p> <pre><code>steps:
- uses: actions/checkout@v6
- uses: actions/setup-python@v6
  with:
    python-version: 'graalpy-24.0'
- run: python my_script.py</code></pre> <p dir="auto"><strong>Free threaded Python</strong></p> <pre><code>steps:
- uses: actions/checkout@v6
- uses: actions/setup-python@v6
  with:
    python-version: '3.13t'
- run: python my_script.py</code></pre> <p dir="auto">The <code>python-version</code> input is optional. If not supplied, the action will try to resolve the version from the default <code>.python-version</code> file. If the <code>.python-version</code> file doesn't exist Python or PyPy version from the PATH will be used. The default version of Python or PyPy in PATH varies between runners and can be changed unexpectedly so we recommend always setting Python version explicitly using the <code>python-version</code> or <code>python-version-file</code> inputs.</p> <p dir="auto">The action will first check the local <a href="https://github.com/actions/setup-python/blob/main/docs/advanced-usage.md#hosted-tool-cache">tool cache</a> for a <a href="https://github.com/npm/node-semver#versions">semver</a> match. If unable to find a specific version in the tool cache, the action will attempt to download a version of Python from <a href="https://github.com/actions/python-versions/releases">GitHub Releases</a> and for PyPy from the official <a href="https://downloads.python.org/pypy/">PyPy's dist</a>.</p> <p dir="auto">For information regarding locally cached versions of Python or PyPy on GitHub hosted runners, check out <a href="https://github.com/actions/runner-images">GitHub Actions Runner Images</a>.</p> <H2 dir="auto">Supported version syntax</H2> <p dir="auto">The <code>python-version</code> input supports the <a href="https://semver.org/">Semantic Versioning Specification</a> and some special version notations (e.g. <code>semver ranges</code>, <code>x.y-dev syntax</code>, etc.), for detailed examples please refer to the section: <a href="https://github.com/actions/setup-python/blob/main/docs/advanced-usage.md#using-the-python-version-input">Using python-version input</a> of the <a href="https://github.com/actions/setup-python/blob/main/docs/advanced-usage.md">Advanced usage</a> guide.</p> <H2 dir="auto">Supported architectures</H2> <p dir="auto">Using the <code>architecture</code> input, it is possible to specify the required Python or PyPy interpreter architecture: <code>x86</code>, <code>x64</code>, or <code>arm64</code>. If the input is not specified, the architecture defaults to the host OS architecture.</p> <H2 dir="auto">Caching packages dependencies</H2> <p dir="auto">The action has built-in functionality for caching and restoring dependencies. It uses <a href="https://github.com/actions/toolkit/tree/main/packages/cache">toolkit/cache</a> under the hood for caching dependencies but requires less configuration settings. Supported package managers are <code>pip</code>, <code>pipenv</code> and <code>poetry</code>. The <code>cache</code> input is optional, and caching is turned off by default.</p> <p dir="auto">The action defaults to searching for a dependency file (<code>requirements.txt</code> or <code>pyproject.toml</code> for pip, <code>Pipfile.lock</code> for pipenv or <code>poetry.lock</code> for poetry) in the repository, and uses its hash as a part of the cache key. Input <code>cache-dependency-path</code> is used for cases when multiple dependency files are used, they are located in different subdirectories or different files for the hash that want to be used.</p> <ul dir="auto"> <li>For <code>pip</code>, the action will cache the global cache directory</li> <li>For <code>pipenv</code>, the action will cache virtualenv directory</li> <li>For <code>poetry</code>, the action will cache virtualenv directories -- one for each poetry project found</li> </ul> <p dir="auto"><strong>Caching pip dependencies:</strong></p> <pre><code>steps:
- uses: actions/checkout@v6
- uses: actions/setup-python@v6
  with:
    python-version: '3.13'
    cache: 'pip' # caching pip dependencies
- run: pip install -r requirements.txt</code></pre> <blockquote> <p dir="auto"><strong>Note:</strong> Restored cache will not be used if the requirements.txt file is not updated for a long time and a newer version of the dependency is available which can lead to an increase in total build time.</p> </blockquote> <blockquote> <p dir="auto">The requirements file format allows for specifying dependency versions using logical operators (for example chardet&gt;=3.0.4) or specifying dependencies without any versions. In this case the pip install -r requirements.txt command will always try to install the latest available package version. To be sure that the cache will be used, please stick to a specific dependency version and update it manually if necessary.</p> </blockquote> <blockquote> <p dir="auto">The <code>setup-python</code> action does not handle authentication for pip when installing packages from private repositories. For help, refer <a href="https://pip.pypa.io/en/stable/topics/vcs-support/">pip’s VCS support documentation</a> or visit the <a href="https://github.com/pypa/pip">pip repository</a>.</p> </blockquote>
