<article><H2 dir="auto">setup-go</H2> <p dir="auto"><a href="https://github.com/actions/setup-go/actions/workflows/basic-validation.yml"><img src="https://github.com/actions/setup-go/actions/workflows/basic-validation.yml/badge.svg" alt="Basic validation"></a> <a href="https://github.com/actions/setup-go/actions/workflows/versions.yml"><img src="https://github.com/actions/setup-go/actions/workflows/versions.yml/badge.svg" alt="Validate 'setup-go'"></a></p> <p dir="auto">This action sets up a Go environment for use in GitHub Actions by:</p> <ul dir="auto"> <li>Optionally downloading and caching a version of Go by version and adding it to the PATH</li> <li>Optionally caching Go modules and build outputs</li> <li>Registering problem matchers for error output</li> </ul> <H2 dir="auto">Breaking changes in V6</H2> <p dir="auto">The V6 edition of the action includes:</p> <ul dir="auto"> <li> <p dir="auto"><strong>Upgraded Node.js runtime from node20 to node24</strong></p> <blockquote> <p dir="auto">Make sure your runner is on version v2.327.1 or later to ensure compatibility with this release. See <a href="https://github.com/actions/runner/releases/tag/v2.327.1">Release Notes</a></p> </blockquote> </li> <li> <p dir="auto"><strong>Go toolchain</strong></p> <ul dir="auto"> <li>Supports both <code>go</code> and <code>toolchain</code> directives in <code>go.mod</code>. If the <code>toolchain</code> directive is present, its version is used; otherwise, the action falls back to the <code>go</code> directive.</li> </ul> </li> <li> <p dir="auto"><strong>Cache key update</strong></p> <ul dir="auto"> <li>By default, the cache key for Go modules is based on <code>go.mod</code>. To use <code>go.sum</code>, configure the <code>cache-dependency-path</code> input.</li> </ul> </li> </ul> <p dir="auto">See full release notes on the <a href="https://github.com/actions/setup-go/releases">releases page</a>.</p> <H2 dir="auto">Usage</H2> <p dir="auto">See <a href="https://github.com/actions/setup-go/blob/main/action.yml">action.yml</a>.</p> <pre><code>- uses: actions/setup-go@v6
  with:
    # Version or version range of Go to use
    go-version: '1.23'

    # Path to go.mod, go.work, .go-version, or .tool-versions file
    # Note: if both go-version and go-version-file are provided, go-version takes precedence.
    go-version-file: 'go.mod'

    # Set this option if you want the action to check for the latest available version
    # Default: false
    check-latest: false

    # GitHub token for authentication
    token: ${{ github.token }}

    # Used to specify whether caching is needed.
    # Default: true
    cache: true

    # Path to dependency files for caching
    cache-dependency-path: 'go.sum'

    # Architecture to install (auto-detected if not specified)
    architecture: 'x64'

    # Custom base URL for Go downloads (e.g., for mirrors)
    go-download-base-url: ''</code></pre> <p dir="auto"><strong>Basic:</strong></p> <pre><code>steps:
  - uses: actions/checkout@v6
  - uses: actions/setup-go@v6
    with:
      go-version: '1.25' # The Go version to download (if necessary) and use.
  - run: go run hello.go</code></pre> <p dir="auto"><strong>Version resolution behavior:</strong></p> <p dir="auto">The action resolves the requested version in the following order:</p> <ol dir="auto"> <li><strong>Local cache</strong> - Checks the local tool cache for a matching <a href="https://github.com/npm/node-semver#versions">semver</a> version.</li> <li><strong>go-versions repository</strong> - If the requested version isn’t available in the tool cache, it pulls the version manifest from the <code>main</code> branch of the <a href="https://github.com/actions/go-versions/blob/main/versions-manifest.json">go-versions repository</a>.</li> <li><strong>Direct download</strong> - If that lookup misses or fails, it will fall back to downloading directly from the <a href="https://go.dev/dl">official Go distribution site</a>.</li> </ol> <p dir="auto">To change the default behavior, please use the <a href="https://github.com/actions/setup-go/blob/main/docs/advanced-usage.md#check-latest-version">check-latest input</a>.</p> <blockquote> <p dir="auto"><strong>Note</strong>: The <code>setup-go</code> action uses executable binaries built by the Go team and does not build Go binaries from source code.</p> </blockquote> <H2 dir="auto">Supported version syntax</H2> <p dir="auto">The <code>go-version</code> input supports the following syntax:</p> <ul dir="auto"> <li>Specific versions: <code>1.25</code>, <code>1.24.11</code>, <code>1.24.0-rc.1</code>, <code>1.23.0-beta.1</code></li> <li>SemVer version range syntax: <code>^1.25.1</code>, <code>~1.24.1</code>, <code>&gt;=1.25.0-rc.1</code>, <code>&lt;1.25.0</code>, <code>&gt;=1.22.0 &lt;1.24.0</code></li> <li>Aliases: <code>stable</code>, <code>oldstable</code></li> <li>Wildcards: <code>1.25.x</code>, <code>1.x</code></li> </ul> <p dir="auto">For details on Semantic Versioning, see <a href="https://github.com/npm/node-semver">the semver package documentation</a>.</p> <blockquote> <p dir="auto"><strong>Note</strong>: Due to the peculiarities of YAML parsing, it is recommended to wrap the version in single quotation marks:</p> <pre><code>go-version: '1.20'</code></pre> <p dir="auto">The recommendation is based on the YAML parser's behavior, which interprets non-wrapped values as numbers and, in the case of version <code>1.20</code>, trims it down to <code>1.2</code>, which may not be very obvious.</p> </blockquote> <p dir="auto">For more usage examples, please refer to the section: <a href="https://github.com/actions/setup-go/blob/main/docs/advanced-usage.md#using-the-go-version-input">Using go-version input</a> of the <a href="https://github.com/actions/setup-go/blob/main/docs/advanced-usage.md">Advanced usage</a> guide.</p> <H2 dir="auto">Recommended permissions</H2> <p dir="auto">When using the <code>setup-go</code> action in your GitHub Actions workflow, it is recommended to set the following permissions to ensure proper functionality:</p> <pre><code>permissions:
  contents: read # access to check out code and install dependencies</code></pre> <H2 dir="auto">Caching dependency files and build outputs</H2> <p dir="auto">The action includes built-in caching and restoration for Go modules and build outputs. It uses <a href="https://github.com/actions/toolkit/tree/main/packages/cache">toolkit/cache</a> under the hood, but requires less configuration. The <code>cache</code> input is optional, and caching is enabled by default. To disable caching, set <code>cache: false</code>.</p> <p dir="auto">By default, the action looks for <code>go.mod</code> in the repository root and uses its hash as part of the cache key. Use the <code>cache-dependency-path</code> input when you have multiple dependency files, or when they’re located in different subdirectories. This input supports glob patterns.</p> <p dir="auto">If caching cannot be performed for any reason, the action logs a warning and continues workflow execution.</p> <p dir="auto">For examples of using <code>cache-dependency-path</code>, see the <a href="https://github.com/actions/setup-go/blob/main/docs/advanced-usage.md#caching">Caching</a> section of the <a href="https://github.com/actions/setup-go/blob/main/docs/advanced-usage.md">Advanced usage</a> guide.</p> <H2 dir="auto">Advanced usage</H2> <ul dir="auto"> <li><a href="https://github.com/actions/setup-go/blob/main/docs/advanced-usage.md#using-the-go-version-input">Using the go-version input</a></li> <li><a href="https://github.com/actions/setup-go/blob/main/docs/advanced-usage.md#using-the-go-version-file-input">Using the go-version-file input</a></li> <li><a href="https://github.com/actions/setup-go/blob/main/docs/advanced-usage.md#check-latest-version">Check latest version</a></li> <li><a href="https://github.com/actions/setup-go/blob/main/docs/advanced-usage.md#caching">Caching</a></li> <li><a href="https://github.com/actions/setup-go/blob/main/docs/advanced-usage.md#outputs">Outputs</a></li> <li><a href="https://github.com/actions/setup-go/blob/main/docs/advanced-usage.md#custom-download-url">Custom download URL</a></li> <li><a href="https://github.com/actions/setup-go/blob/main/docs/advanced-usage.md#using-setup-go-on-ghes">Using <code>setup-go</code> on GHES</a></li> </ul> <H2 dir="auto">License</H2> <p dir="auto">The scripts and documentation in this project are released under the <a href="https://github.com/actions/setup-go/blob/main/LICENSE">MIT License</a>.</p> <H2 dir="auto">Contributions</H2> <p dir="auto">Contributions are welcome!
